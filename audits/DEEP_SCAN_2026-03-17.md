## P0 — Non-Negotiable Violations & Production Risk

### [SCAN-001] XSS in server-rendered practitioner profile HTML
- **Principle violated:** Security / OWASP
- **File:** `workers/src/handlers/practitioner-profile.js:47-128`
- **What's wrong:** User-controlled `display_name` and `bio` are interpolated into server-rendered HTML (OG tags, `<title>`, `<h1>`, `<p>`) with zero HTML escaping. A practitioner can inject arbitrary HTML/JS via their bio field.
- **Fix direction:** HTML-escape all user data before interpolation. Create an `escapeHtml()` utility in workers/src/lib/.

### [SCAN-002] Unauthenticated email unsubscribe endpoint
- **Principle violated:** Security / Auth
- **File:** `workers/src/index.js:415-424`
- **What's wrong:** `POST /api/email/unsubscribe` accepts an email in the body and runs an UPDATE query with no auth, no verification token, no rate limit. An attacker can unsubscribe any user.
- **Fix direction:** Require a signed unsubscribe token (generated per-email) or require auth. Add rate limiting.

### [SCAN-003] Synthesis SYSTEM_PROMPT instructs AI to state identity as fact
- **Principle violated:** Non-Negotiable 2 (AI never presents interpretation as fact)
- **File:** `src/prompts/synthesis.js:33`
- **What's wrong:** "You are constructing the mirror in which they recognize what they already are." The `whoYouAre` schema (line 258) says "IDENTITY — name the core quality that makes them distinct, stated as fact not label." Examples use "You are wired to initiate things others won't."
- **Fix direction:** Reframe SYSTEM_PROMPT as pattern-based lens. Change whoYouAre instruction to "stated as a pattern their data reveals, not a fixed identity."

### [SCAN-004] Specificity examples in synthesis.js contradict grounding rules
- **Principle violated:** Non-Negotiable 2
- **File:** `src/prompts/synthesis.js:196`
- **What's wrong:** GROUNDING RULES say "this suggests not this means" but examples say "you catch subtle social dynamics" and "your body gives you instant 'yes/no' hits" as absolute statements.
- **Fix direction:** Rewrite examples to use "your data suggests" / "the pattern indicates" framing.

### [SCAN-005] Frontend type/authority guidance uses verdict language
- **Principle violated:** Non-Negotiable 2
- **File:** `frontend/js/app.js:2969-2996`
- **What's wrong:** Static `typeGuidance`, `authorityGuidance`, `definitionGuidance` objects use "As a Generator, your life force is the most abundant…", "You are built to master things…" — absolute identity verdicts.
- **Fix direction:** Rewrite all guidance entries using pattern-based framing: "Your chart shows a Builder Pattern, which tends toward…"

### [SCAN-006] "Who You Are" section heading frames AI output as identity statement
- **Principle violated:** Non-Negotiable 2
- **File:** `frontend/js/app.js:3291`
- **What's wrong:** Heading says "Who You Are" with no disclaimer that this is an interpretation. Combined with SCAN-003, the AI is told to present fact, and the UI labels it as identity.
- **Fix direction:** Rename to "What Your Data Reveals" or add inline disclaimer before the section.

### [SCAN-007] Life cycle guidance uses prophetic verdict language
- **Principle violated:** Non-Negotiable 2
- **File:** `src/engine/transits.js:659,674-675`
- **What's wrong:** `getCycleGuidance()` returns "You are now the wounded healer", "You are reborn. Claim your power." These are prophetic statements of fact about a person's life.
- **Fix direction:** Reframe as pattern: "This period's data aligns with themes of healing and integration."

### [SCAN-008] Cluster synthesis prompt lacks "lens/pattern" framing guardrails
- **Principle violated:** Non-Negotiable 2
- **File:** `workers/src/handlers/cluster.js:579-589`
- **What's wrong:** The cluster LLM prompt says "Analyze the group composition and provide actionable guidance" but has no instruction to frame output as pattern-based rather than factual. The LLM could produce "Member 2 is the natural leader" as fact.
- **Fix direction:** Add GROUNDING RULES equivalent to cluster prompt: "Frame all observations as patterns the data suggests, not facts about the individuals."

### [SCAN-009] SMS/weekly digest prompts have no lens framing
- **Principle violated:** Non-Negotiable 2
- **File:** `src/prompts/digest.js:27-37,74-84`
- **What's wrong:** Both digest prompts instruct the LLM to reference chart data but include zero instruction to frame output as interpretation. An LLM could say "Today your Sacral says yes" as fact.
- **Fix direction:** Add "Frame all guidance as patterns your chart suggests, not definitive inner experiences" to the prompt rules.

### [SCAN-010] Design solver silently returns non-converged result
- **Principle violated:** Non-Negotiable 1 (Calculation integrity)
- **File:** `src/engine/design.js:60-62`
- **What's wrong:** Newton-Raphson solver returns best estimate on convergence failure with no error flag. All downstream gate/line calculations proceed on potentially wrong JDN.
- **Fix direction:** Throw an error or return a `{ converged: false }` flag that the caller must check. Never silently proceed with unverified data.

### [SCAN-011] 7 engine layers return `{error}` objects not checked by orchestrator
- **Principle violated:** Non-Negotiable 1
- **Files:** `src/engine/chiron.js:57`, `lilith.js:71`, `vedic.js:255`, `bazi.js:325`, `mayan.js:141`, `sabian.js:99`, `ogham.js:99`; orchestrator at `src/engine/index.js:126`
- **What's wrong:** Each catch returns `{ error: err.message }`. The orchestrator never checks for `.error` property — passes error objects into chart data → RAG context → LLM prompt.
- **Fix direction:** Add error checks in index.js orchestrator. Either throw or exclude errored layers from the chart with a visible flag.

### [SCAN-012] RAG context builder silently returns empty string on any error
- **Principle violated:** Non-Negotiable 1
- **File:** `src/prompts/rag.js:714`
- **What's wrong:** Entire `buildRAGContext()` in try/catch returns `''` on failure. LLM synthesizes profile with zero knowledgebase grounding data, no warning.
- **Fix direction:** Return a partial context with a `ragError` flag, or throw so the caller can decide how to handle it.

### [SCAN-013] 49 forbidden vocabulary terms in user-facing strings
- **Principle violated:** Principle 6 §8 (Approved Vocabulary)
- **Files:** `frontend/js/app.js` (20), `workers/src/handlers/composite.js` (9), `workers/src/lib/email.js` (7), `workers/src/handlers/practitioner-profile.js` (4), `workers/src/handlers/cluster.js` (5), `frontend/components/ShareCard.js` (1), `frontend/components/AIContextEditor.js` (1), `workers/src/handlers/practitioner.js` (1), `workers/src/handlers/onboarding.js` (1)
- **What's wrong:** "Human Design", "Bodygraph", "Incarnation Cross", "Not-Self Theme", "Split Definition", "Generator/Projector/Manifestor/Reflector", "Sacral/Splenic/Emotional/Self-Projected/Ego Authority", "Shadow" (bare, in GK context) all appear in user-facing strings and AI prompts. See Step 1 findings for full line-by-line inventory (51 total incl. 2 P1).
- **Fix direction:** Create a `mapDisplayName()` utility that converts engine terms to canonical vocabulary. Apply at rendering time, not in the engine/DB.

### [SCAN-014] Agency pricing claims unbuilt features (False Upgrade)
- **Principle violated:** Principle 10 (Honest Grounding) / Non-Negotiable 3
- **Files:** `frontend/index.html:433`, `frontend/pricing.html:155`
- **What's wrong:** Pricing table lists "White-label client portal" for Agency at $349/mo. Code does not support this (billing.js:121 acknowledges it). Agency checkout gates via `contactRequired: true` but the pricing page does not communicate this is unavailable.
- **Fix direction:** Remove unbuilt features from pricing table, or add "Coming soon" labels. Do not list capabilities the code does not support.

### [SCAN-015] Wrong quota numbers shown to users (stale tierLimits)
- **Principle violated:** Principle 6 (One Truth, One Source) / Principle 10 (Honest Grounding)
- **File:** `frontend/js/app.js:7308-7316`
- **What's wrong:** Frontend `tierLimits` says `regular: 30, practitioner: 200, white_label: 1000`. Backend `getTierConfig()` says `individual: 10, practitioner: 500, agency: 2000`. The `individual` key doesn't even exist in the frontend map. Users see wrong quota information.
- **Fix direction:** Remove frontend tierLimits. Fetch quotas from the backend or derive from `/api/auth/me` response.

### [SCAN-016] Life cycle dates use linear period approximation
- **Principle violated:** Non-Negotiable 1
- **File:** `src/engine/transits.js:528-529`
- **What's wrong:** Saturn Return calculated as `birthJDN + (29.46 * N * 365.25)` — linear estimate, not exact planetary position. Can be off by months. Output includes `approximate: true` but user sees no warning.
- **Fix direction:** Either calculate exact returns using iterative solver, or add visible "approximate" label in the UI when displaying life cycle dates.
