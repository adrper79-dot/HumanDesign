The QUERIES.getLatestChart query doesn't exist or has wrong column name
The chart JSON parsing throws

CREATE TABLE practitioner_invitations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  practitioner_id UUID NOT NULL REFERENCES practitioners(id) ON DELETE CASCADE,
  client_email TEXT NOT NULL,
  client_name TEXT,
  token TEXT UNIQUE NOT NULL,         -- secure random token
  status TEXT DEFAULT 'pending'       -- pending | accepted | expired
    CHECK (status IN ('pending','accepted','expired')),
  message TEXT,                        -- optional personal message from practitioner
  expires_at TIMESTAMPTZ NOT NULL DEFAULT (NOW() + INTERVAL '14 days'),
  accepted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_invitations_practitioner ON practitioner_invitations(practitioner_id);
CREATE UNIQUE INDEX idx_invitations_pract_email 
  ON practitioner_invitations(practitioner_id, client_email) 
  WHERE status = 'pending';

4.3 Bulk Profile Generation

Endpoint: POST /api/practitioner/clients/bulk-profiles — body: { clientIds: [uuid, uuid, ...] }

Processing: Sequential with 500ms gap (to avoid LLM rate limits). Returns streaming progress: { processed: 3, total: 10, currentClient: "Jane" }.

Quota handling: Each generation counts against practitioner's profileGenerations quota (200/month on Guide tier). The UI shows remaining quota before bulk operation.

4.4 Client Offboarding
When a practitioner removes a client from their roster:

Client's data stays (it's theirs)
Client portal link is revoked
Session notes are archived (practitioner retains read-only access to their own notes)
Client is notified by email: "[Practitioner] has removed you from their client roster. Your Prime Self account and chart are not affected."
This is professional. No HD tool handles this gracefully — practitioners currently just stop responding, which is an awful experience for clients.
