WHAT IS BROKEN RIGHT NOW (BLOCKING REVENUE)
These are not roadmap items. These are fires.

FIRE 1: Stripe Checkout Fails in Firefox
File: frontend/index.html ~line 1210
Impact: 100% of Firefox users cannot upgrade. Firefox is ~8% of desktop browser market. Every Firefox user who hits the upgrade modal gets a silent JavaScript crash (TypeError: Cannot read property 'target' of undefined). They don't know why it failed. They leave.
Fix time: 10 minutes. startCheckout(tier) needs startCheckout(tier, event) with the onclick passing event.

FIRE 2: Practitioners Cannot Load Client Charts
File: workers/src/handlers/profile.js — GET /api/chart/:id
Impact: The entire Guide tier value proposition is broken. A practitioner who upgrades to $60/mo and tries to open a client chart gets a 500. This is the core workflow. If any practitioner has tried the product and hit this, they churned immediately and told others.
Fix time: 2–4 hours to diagnose and fix the 500.

FIRE 3: Password Reset Returns 500
Files: workers/src/handlers/auth.js
Impact: Users who forget their password cannot recover. They cannot come back. Migration 021 not applied to production. Every password reset attempt fails silently.
Fix time: Apply migration. 15 minutes.

FIRE 4: Pull-to-Refresh Is Fake
File: frontend/index.html ~line 4156–4185
Impact: The gesture is detected, a spinner shows, but only console.log('🔄 Refreshing...') runs. No data reloads. Users pull down, see the spinner, and nothing changes. On mobile this is the primary way people expect to refresh. It trains users that the app doesn't work properly.
Fix time: 30 minutes.

FIRE 5: XSS in Check-in Save
File: frontend/index.html ~line 4241, 4265, 4332
Impact: API error messages inserted directly into innerHTML without escaping. If any API error contains HTML/JS, it executes.
Fix time: 15 minutes. Wrap with escapeHtml().

FIRE 6: PWA Offline Is Completely Broken
File: offline-transits.js ~line 147, 191, 243
Impact: The service worker is registered. Users may believe offline mode works. It doesn't. The code checks .ok on a parsed JSON object, which never has .ok — so prefetching silently fails every time. The PWA badge is lying.
Fix time: 15 minutes to fix the .ok → error-check pattern.
